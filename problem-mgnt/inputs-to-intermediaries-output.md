## 7. Pipeline Flow Summary

```
Raw Excel (raw/)
    │
    ▼
[Human analysis + bringyourskill_harness.md template]
    │
    ├─► pm-uc-brief.md              (L1 — what & why)
    ├─► pm-workflow-spec.md         (L2 — how, phase by phase)
    └─► pm-skill-spec-sk06-*.md    (L3 — new skill contract)
            │
            ▼
    [generate_harness.py / generate_skill_lambda.py]  ← future step
            │
            ├─► Lambda handler code (code/lambda/harness/pm_harness/)
            ├─► Lambda handler code (code/lambda/skills/problem_classifier/)
            ├─► Terraform resource blocks (harness_generated.tf or pm_generated.tf)
            └─► Test stubs (run_generated_tests.py)
            │
            ▼
    [deploy.sh → terraform apply + docker build/push]
            │
            ▼
    AWS: PM harness Lambda + SK-06 Lambda + PM S3 bucket
            │
            ▼
    Streamlit agent selector → "Problem Management (UC-PM)"
            → reads PM_WIKI_BUCKET env → llmwiki-problem-mgnt-e0a2165b
```

## 9. Prerequisite: Bedrock Model Access

All Bedrock-backed Lambdas (harness + all skills) require Anthropic model EULA acceptance.

**Action required (AWS Console only — cannot be done via CLI from member account):**

1. Open [https://console.aws.amazon.com/bedrock/home?region=us-east-1#/modelaccess](https://console.aws.amazon.com/bedrock/home?region=us-east-1#/modelaccess)
2. Click **"Manage model access"**
3. Select all Anthropic models: Claude Sonnet 4.6, Claude Opus 4.7, Claude Haiku 4.5, Claude Opus 4.6, Claude Sonnet 4.5, Claude Opus 4.5, Claude Opus 4.1
4. Submit use-case form and accept EULA
5. Wait for status to change from `NOT_AVAILABLE` → `AVAILABLE`

Once done, re-run the harness smoke test:
```bash
aws lambda invoke \
  --function-name llmwiki-uc1-harness \
  --payload '{"customer_id":"test-s2s-001","customer_name":"Test Health Plan","product":"TriZetto QNXT","sow_reference":"SOW-2026-TEST-001","human_context":""}' \
  --cli-binary-format raw-in-base64-out \
  --region us-east-1 --profile tzg-sandbox /tmp/harness_test.json && cat /tmp/harness_test.json
```
